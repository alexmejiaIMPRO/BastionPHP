Quick Start

Get up and running in under 5 minutes

This guide will help you create your first Bastion PHP application. No deep PHP knowledge required.

Prerequisites

  • PHP 8.1 or higher
  • Composer (PHP package manager)
  • SQLite (usually included with PHP)
  • Node.js (optional, for Tailwind)

What you'll build

  • Secure project structure
  • Database with migrations
  • Auth system with JWT
  • Live reload dev environment
Terminal - Create your first app 
# Clone the generator
git clone https://github.com/alexmejiaIMPRO/BastionPHP.git
cd BastionPHP

# Create a new project
bash create-bastion-app MyApp

# Navigate into your project
cd MyApp

# Install dependencies
composer install
npm install

# Set up database
./bastion migrate
./bastion seed

# Start development server
./bastion run-dev

# Your app is now running at:
# http://localhost:9876

First time with PHP?

Don't worry! Bastion PHP is designed to be approachable. Each command is explained below.

Command explanations
composer install

Installs PHP dependencies like the router and database layer.

npm install

Installs Tailwind CSS and BrowserSync for live reload.

./bastion migrate

Creates database tables by running SQL migration files.

./bastion seed

Populates the database with demo data (admin user, etc).

./bastion run-dev

Starts PHP server, Tailwind watcher, and BrowserSync proxy.

Framework Overview

Understanding Bastion PHP's architecture

Bastion PHP is a framework generator that creates secure, production-ready PHP applications with modern development patterns.

Security-First Design

  • • CSP nonces for inline scripts
  • • CSRF protection middleware
  • • JWT authentication with refresh tokens
  • • Secure cookies (HttpOnly, SameSite)
  • • Explicit HTML escaping with e()

Modern Developer Experience

  • • File-based routing (like Next.js)
  • • FastAPI-style endpoints
  • • Layout nesting and composition
  • • Hot reload with BrowserSync
  • • Tailwind v4 integration

Core Philosophy

1

Convention Over Configuration

Sensible defaults mean you spend less time configuring and more time building features.

2

Progressive Disclosure

Start simple with file-based routing, dive deeper when you need custom middleware or advanced DB queries.

3

Security Never Optional

All security features are enabled by default. Opt-out is harder than opt-in, protecting against developer oversight.

Important: Web server configuration

Always point your web server to the public/ directory, not the project root. This prevents exposing sensitive files like .env.

Filesystem Routing

Next.js-inspired file-based routing

Bastion PHP uses your folder structure as the routing system. No configuration required.

Pages

Page Routes

app/page.php → /
app/about/page.php → /about
app/admin/users/page.php → /admin/users
API

API Routes

api/users/+server.php → /api/users
api/users/[id]/+server.php → /api/users/123
api/auth/+server.php → /api/auth/*
Example: app/admin/users/page.php 
<?php
use App\Core\DV;
use App\Core\DB;

// Set page metadata
DV::set('title', 'Admin - User Management');
DV::set('breadcrumbs', ['Admin', 'Users']);

// Fetch data
$users = DB::table('users')
  ->orderBy('created_at', 'DESC')
  ->limit(50)
  ->get();
?>
