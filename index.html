The Bastion Guide

Bastion is an enterprise-grade PHP framework designed to standardize development across organizations. It combines the Next.js App Router experience with battle-tested PHP technologies.

Secure by Default

Path traversal protection, CSRF, CSP nonces, session regeneration, and more enabled out of the box.

File-Based Routing

Organize your app with intuitive folder structure. No route definitions needed. Just like Next.js.

Global Styling

One config file controls styling for the entire app. Switch between Tailwind CDN, Build, or Custom CSS instantly.

Quick Start

Terminal 
# 1. Create New Project
bash create-php-app.sh my-app --with-auth

# 2. Install Dependencies
cd my-app
composer install
npm install

# 3. Setup Database
php artisan migrate
php artisan db:seed

# 4. Start Development Server
php artisan serve
Development Port Bastion PHP uses port 9876 by default. Visit http://127.0.0.1:9876.

The app/ Directory

Contains your routing logic, views, and functional components. The folder structure corresponds directly to the URL paths.

The core/ Directory

Contains the engine: Router.php, Auth.php, DB.php, etc. You rarely need to touch this unless extending the framework.

GS

Global Styling System

Centralized styling management. No manual <link> tags needed.

One of Bastion PHP's most powerful features. The Global Styling System automatically injects the correct CSS/JavaScript into your pages based on a single configuration file. It supports Tailwind (CDN or Build mode) and Custom CSS out of the box.

Configuration

config/style.php 
return [
    // Enable/disable Tailwind CSS globally
    'useTailwind' => true,

    // 'cdn' (Dev) or 'build' (Production)
    'tailwindMode' => 'cdn',

    // Fallback if useTailwind is false
    'fallbackCss' => '/css/fallback.css',
];

Helper Functions

Any .php view 
// Check if Tailwind is active
if (use_tailwind()) {
    echo '<div class="bg-blue-500">';
}

// Check mode ('cdn' or 'build')
$mode = tailwind_mode();

// Get any style config
$fallback = style_config('fallbackCss');

Tailwind Modes Explained

Mode 1: CDN (Development)

Best for local development and rapid prototyping.

  • Loads Tailwind via CDN script.
  • JIT (Just-In-Time) compilation in browser.
  • No build step required.
  • Slightly slower initial load (~200ms).

Mode 2: Build (Production)

Best for production deployments.

  • Loads pre-compiled CSS from /public/css/app.css.
  • Zero runtime overhead.
  • Requires npm run build.
  • Optimized and purged CSS.

Routing & Layouts

Bastion PHP uses file-based routing. Drop a page.php file anywhere in your app/ directory and it becomes a route.

File Path URL Params
app/page.php / -
app/about/page.php /about -
app/users/[id]/page.php /users/123 $params['id']
app/blog/[slug]/page.php /blog/hello-world $params['slug']

Nested Layouts

Layouts wrap your pages. Bastion supports nested layouts, meaning a page is rendered inside its parent folder's layout, which is rendered inside its parent, up to the root.

app/layout.php (Root) 
<!doctype html>
<html>
<body>
    <nav>Main Navigation</nav>
    
    <!-- Child content injected here -->
    <?= $content ?>
    
    <footer>Main Footer</footer>
</body>
</html>
app/dashboard/layout.php (Nested) 
<div class="dashboard-grid">
    <aside>Sidebar</aside>
    
    <main>
        <!-- Page content injected here -->
        <?= $content ?>
    </main>
</div>

API Routes

API endpoints are defined using +server.php files. These files return an array of closures keyed by HTTP method.

app/api/users/[id]/+server.php DYNAMIC ROUTE 
use App\Core\Request;
use App\Core\Response;
use App\Models\User;

return [
    // GET /api/users/123
    'get' => function (Request $req) {
        $id = $req->meta['params']['id'];
        
        $user = User::find($id);
        
        if (!$user) {
            Response::json(['error' => 'Not found'], 404);
        }
        
        Response::json($user);
    },

    // DELETE /api/users/123
    'delete' => function (Request $req) {
        $id = $req->meta['params']['id'];
        User::delete($id);
        
        Response::json(['success' => true]);
    }
];

Components & HTMX

Bastion PHP encourages using Functional Components instead of complex template engines. Combine PHP functions with Heredoc syntax and HTMX for a React-like experience with zero build steps.

1. Define Component

app/components/UserCard.php
function UserCard($user) {
    $name = e($user['name']);
    $id = $user['id'];
    
    return <<<HTML
    <div class="card" id="user-$id">
        <h3>$name</h3>
        <button 
            hx-delete="/api/users/$id"
            hx-target="#user-$id"
            hx-swap="outerHTML">
            Delete
        </button>
    </div>
    HTML;
}

2. Use in Page

app/users/page.php
require_once '../components/UserCard.php';

$users = User::all();
?>

<div class="grid">
    <?php foreach ($users as $user): ?>
        <?= UserCard($user) ?>
    <?php endforeach; ?>
</div>
Auth

Authentication (JWT)

Secure Access & Refresh Token implementation.

Bastion includes a complete JWT authentication system. It uses short-lived Access Tokens (15 min) for requests and long-lived HttpOnly Refresh Tokens (7 days) for session maintenance.

Issuing Tokens (Login)

use App\Core\Auth;

$tokens = Auth::issueTokens($userId);

// Set Secure HttpOnly Cookie
setcookie('refresh', $tokens['refresh'], [
    'expires' => time() + 604800,
    'path' => '/',
    'httponly' => true,
    'secure' => true,
    'samesite' => 'Lax'
]);

Protecting Routes

// In page.php
if (!auth()) {
    redirect('/login');
}

// In API +server.php
return [
    'get' => function($req) {
        Auth::requireAuth($req, function() {
            // Protected logic
        });
    }
];

Database & Models

Bastion uses SQLite with WAL mode enabled by default for 10-100x faster concurrent writes. MySQL and PostgreSQL are also supported via .env configuration.

Fluent Query Builder

$users = DB::table('users')
    ->where('role', 'admin')
    ->orderBy('created_at', 'DESC')
    ->limit(10)
    ->get();

$id = DB::table('users')->insert([
    'email' => 'jane@example.com',
    'active' => 1
]);

Data Models

namespace App\Models;
use App\Core\DB;

class User {
    public static function find($id) {
        return DB::pdo()
            ->prepare("SELECT * FROM users WHERE id=?")
            ->execute([$id])
            ->fetch();
    }
}

Deployment

Standard Deployment Script 
# 1. Clone & Install
git clone https://github.com/your-org/app.git
cd app
composer install --no-dev --optimize-autoloader

# 2. Permissions
chmod 755 storage/
chmod 664 storage/db/app.db

# 3. Environment
cp .env.example .env
php artisan key:generate
# Edit .env for production DB credentials

# 4. Build Assets (Tailwind)
npm install
npm run build

# 5. Migrate
php artisan migrate

Web Server Configuration

Point your web server document root to the public/ folder.

Nginx

server {
    root /var/www/app/public;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }
}

Apache

Bastion includes a .htaccess file in the public directory that handles URL rewriting automatically.

Multi-Plant Deployment

Strategies for deploying standard tools across multiple physical locations (plants) with minimal friction.

Phase 1: Pilot (Week 1-2)

Deploy to a single "tech-savvy" plant to validate the folder structure and dev experience.

./deploy.sh "Plant A" "PL01"

Phase 2: Rollout & Sharing

Use a centralized "Modules" repository. Plants can pull specific modules (e.g., Quality Control, Maintenance) into their app/ folder.

# Installing a shared module
git clone git@github.com:org/modules.git /tmp/modules
cp -r /tmp/modules/quality-control/app/quality app/
php artisan migrate

Styling Consistency

Use config/style.php to manage consistency. Plant A can run in 'Build' mode for stability, while Plant B runs in 'CDN' mode for rapid development of a new feature, using the exact same codebase.
