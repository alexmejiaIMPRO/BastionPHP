Introduction

Bastion PHP v3.0 is an enterprise-grade framework designed for standardization. It combines the Service Container pattern with File-Based Routing and Tailwind CSS v4.

üèõÔ∏è Service Container

Full Dependency Injection and Singleton management.

‚ö° Process Manager

New ./bastion CLI handles PHP, Tailwind, and BrowserSync concurrently.

üëÅÔ∏è DV Engine

Explicit Data View engine. No more global variable pollution.

üé® Tailwind v4

Native Oxide engine support. Config lives in CSS, not JS.

Installation

bash create-bastion-app.sh my-app --full-stack

Starting Development

We now use a single command to start the entire environment:

cd my-app
./bastion run-dev

This starts PHP (port 8000), Tailwind Watcher, and BrowserSync (port 9876).

üèõÔ∏è Core Architecture

New in v3.0: We moved from static classes to a Container-based architecture with Facades.

The Container

Services are bound in app/Core/App.php:

$this->singleton(DB::class, fn() => new DB());

Facades

You write clean static code, but it executes via the container instance for testability.

// Clean Syntax
DB::table('users')->get();

// Actual Execution
App::getInstance()->resolve(DB::class)->query('users')->get();

‚ö° Bastion CLI

The ./bastion script is your new best friend. It is a process manager written in PHP.

Commands

  • ./bastion run-dev: Starts Dev Server + Watchers + Live Reload.
  • ./bastion run-build: Compiles and minifies CSS for production.

üé® Global Styling (Tailwind v4)

Bastion v3 uses Tailwind CSS v4 (Oxide). Configuration is now done in CSS.

Configuration (resources/css/app.css)

@import "tailwindcss";

@variant dark (&:where(.dark, .dark *));

@theme {
  --font-sans: "Inter", system-ui, sans-serif;
  --color-brand: #3b82f6;
}

Theme Helper

Use Theme::apply() in your layouts. It handles system preference detection automatically.

<html lang="en" <?= \App\Core\Theme::apply() ?>>

üëÅÔ∏è DV (Data View) Engine

Stop using global $title. Use the DV engine to pass data explicitly.

Setting Data

<?php
use App\Core\DV;

// Pass data to the layout and view
DV::set('title', 'Dashboard');
DV::set('user', Auth::user());
?>

Using Data

The view() helper extracts these variables locally.

<h1><?= $title ?></h1>

File-Based Routing

Create folders in app/. Use page.php for views and +server.php for API.

FileURL
app/page.php/
app/users/[id]/page.php/users/123

Accessing Parameters

Dynamic parameters are automatically injected into DV.

<?php
$id = $params['id'];
?>
<h1>User <?= $id ?></h1>

Deployment

  1. Clone repo to server.
  2. Run composer install --no-dev.
  3. Run ./bastion run-build (Compiles CSS).
  4. Point web server to public/index.php.
